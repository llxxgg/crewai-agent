# LLM 参数配置最佳实践与反模式

- temperature 固定格式使用0.2以下，发散用0.8以上
```markdown
Temperature (0.0 to 1.0) controls response randomness. Lower values (e.g., 0.2) produce more focused, deterministic 
outputs, while higher values (e.g., 0.8) increase creativity and variability.
```
- top_k 与temperature配合，不建议两个参数一起调整
- timeout
- max_retries
- max_tokens
- stop 当输出什么token立即停止，注意bad case
- stream 是否流式，按需开启会影响连接数
